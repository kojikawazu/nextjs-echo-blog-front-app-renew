# 機能仕様書（Functional Specification）

## 1. ホーム画面（`/`）

### 画面構成

```
┌─────────────────────────────────────────────┐
│ Header（ロゴ、ナビゲーション、認証ボタン）       │
├──────────────────────┬──────────────────────┤
│ Main Content          │ Sidebar              │
│ ┌──────────────────┐ │ ┌──────────────────┐ │
│ │ BlogFilter       │ │ │ カテゴリ一覧      │ │
│ │ (検索/フィルタ)   │ │ ├──────────────────┤ │
│ ├──────────────────┤ │ │ 人気記事TOP5      │ │
│ │ BlogCard × N     │ │ ├──────────────────┤ │
│ │ (記事カード一覧)  │ │ │ タグ一覧          │ │
│ ├──────────────────┤ │ └──────────────────┘ │
│ │ Pagination       │ │                      │
│ └──────────────────┘ │                      │
├──────────────────────┴──────────────────────┤
│ Footer                                       │
└─────────────────────────────────────────────┘
```

### BlogFilter 操作仕様

| 操作 | 動作 |
|------|------|
| テキスト入力（検索ボックス） | 300msデバウンス後、タイトル・説明文・タグで絞り込み |
| カテゴリ選択 | 選択したカテゴリの記事のみ表示 |
| タグ選択 | 選択したタグを含む記事のみ表示 |
| ソート切替（newest/popular） | 新着順またはいいね数順にソート |

### ページネーション

- 1ページあたり10件
- クライアントサイドでページング処理（全件取得後にスライス）
- **既知の制限**: フィルタリング・検索・ソート変更時にページが1にリセットされない（`currentPage` の初期化処理が未実装）

### データフロー

```
Home コンポーネント
  → useQuery(['blogs', {filters}])
    → fetchBlogs(page, limit, tag, category, sortBy, searchQuery)
      → GET /api/blogs
        → proxyToBackend('/blogs')
          → バックエンドAPI GET /blogs
```

## 2. ブログ詳細画面（`/blog/[id]`）

### 画面構成

```
┌─────────────────────────────────────────────┐
│ Header                                       │
├──────────────────────┬──────────────────────┤
│ Article               │ Sidebar              │
│ ┌──────────────────┐ │                      │
│ │ 作成日 / 編集ボタン│ │                      │
│ │ タイトル          │ │                      │
│ │ タグ一覧          │ │                      │
│ │ 説明文            │ │                      │
│ │ Markdown本文      │ │                      │
│ ├──────────────────┤ │                      │
│ │ CommentsSection  │ │                      │
│ │ (コメント欄)     │ │                      │
│ └──────────────────┘ │                      │
├──────────────────────┴──────────────────────┤
│ Footer                                       │
└─────────────────────────────────────────────┘
```

### Markdownレンダリング仕様

1. `fetchBlogById(id)` でブログデータを取得
2. ブログに `github_url` が設定されている場合:
   - `POST /api/github/markdown` でMarkdownを取得
   - gray-matterでフロントマター除去
   - ReactMarkdownでレンダリング
3. プラグイン: remark-gfm（テーブル等）、remark-breaks（改行）、rehype-highlight（シンタックスハイライト）
4. CSSテーマ: atom-one-dark（highlight.js）

### 認証状態による表示切替

| 条件 | 表示 |
|------|------|
| 未認証 | 記事閲覧 + コメント投稿のみ |
| 認証済み & 記事オーナー | 記事閲覧 + 「編集」リンク |
| 認証済み & 非オーナー | 記事閲覧のみ |

## 3. 記事作成画面（`/new`）

### アクセス制御

- 未認証ユーザー → ホーム（`/`）へリダイレクト
- 認証読み込み中 → ローディングスピナー表示

### フォーム項目

| 項目 | フィールド名 | バリデーション | 必須 |
|------|-------------|---------------|------|
| タイトル | title | 1文字以上（Zodはoptional、HTMLのrequired属性で必須化） | ○ |
| GitHub URL | github_url | URL形式（Zodはoptional、HTMLのrequired属性で必須化） | ○ |
| カテゴリ | category | 1文字以上（Zodはoptional、HTMLのrequired属性で必須化） | ○ |
| タグ | tags | 1文字以上（Zodはoptional、HTMLのrequired属性で必須化） | ○ |
| 内容 | description | 1文字以上（Zodはoptional、HTMLのrequired属性で必須化） | ○ |

> **注意**: Zodスキーマ上は全フィールドが `optional()` であり、必須チェックはHTMLの `required` 属性に依存している。

### 操作フロー

```
フォーム入力 → Zodバリデーション → 確認モーダル表示 → 「作成」クリック
  → POST /api/blogs → 成功: toast + ホームへリダイレクト
                     → 失敗: toastエラー
```

## 4. 記事編集画面（`/edit/[id]`）

### アクセス制御

| 条件 | 動作 |
|------|------|
| 未認証 | `/login` へリダイレクト |
| 認証済み & 非オーナー | `/login` へリダイレクト |
| ブログデータ取得失敗 | `/` へリダイレクト |

### 操作フロー（更新）

```
既存データ表示 → フォーム編集 → Zodバリデーション → 更新確認モーダル
  → PUT /api/blogs/:id → 成功: toast + 記事詳細へリダイレクト
                        → 失敗: toastエラー
```

### 操作フロー（削除）

```
「削除」ボタン → 削除確認モーダル
  → DELETE /api/blogs/:id → 成功: toast + ホームへリダイレクト
                           → 失敗: toastエラー
```

## 5. ログイン画面（`/login`）

### フォーム項目

| 項目 | フィールド名 | バリデーション |
|------|-------------|---------------|
| メールアドレス | email | 有効なメールアドレス形式 |
| パスワード | password | 6文字以上 |

### 操作フロー

```
フォーム入力 → Zodバリデーション → POST /api/auth/login
  → 成功: toast「ログインしました」+ 認証状態再取得 + ホームへリダイレクト
  → 失敗: toast「メールアドレスまたはパスワードが正しくありません」
```

## 7. コメント機能仕様

### コメント投稿

- ゲストユーザー名（guest_user）とコメント本文（comment）を入力
- Zodバリデーション適用
- 投稿成功後、TanStack Queryのキャッシュ無効化で一覧を再取得
- APIリクエストボディはcamelCase（`blogId`, `guestUser`, `comment`）

### リプライ機能

> **未実装**: 型定義に `parent_id` フィールドが存在するが、リプライUI・`parent_id` の送信ロジック・スレッド表示は全て未実装。コメントはフラットなリスト表示のみ。

## 8. いいね機能仕様

### 訪問者ID管理

1. 初回アクセス時: `GET /api/blog-likes/generate-visit-id` で訪問者ID生成
2. **既知の制限**: 生成された訪問者IDは永続化されていない。`useLikeBlog` フック内で `generateVisitId()` を呼び出すが、返却値を保存せず `NEXT_PUBLIC_VISIT_ID_KEY`（env変数のキー名）をstateに入れるのみで未使用。訪問者IDの管理はバックエンドのCookieに依存している可能性がある。

### いいね操作

| 操作 | APIコール | 動作 |
|------|----------|------|
| いいね | POST /api/blog-likes/:blogId | いいね登録 + キャッシュ無効化 |
| いいね取消 | DELETE /api/blog-likes/:blogId | いいね削除 + キャッシュ無効化 |

### いいね済み判定

- `GET /api/blog-likes` でいいね済みブログID一覧を取得
- `hasLiked(blogId)` で判定

## 9. 共通レイアウト

### Header

| 要素 | 未認証時 | 認証時 |
|------|---------|--------|
| ロゴ（TechBlog） | ○ | ○ |
| 「記事一覧」リンク | ○ | ○ |
| 「新規投稿」リンク | × | ○ |
| ユーザー名 | × | ○ |
| 「ログアウト」ボタン | × | ○ |
| 「ログイン」ボタン | ○ | × |

### Sidebar

- カテゴリ一覧: GlobalContextから取得、クリックで `/category/:category` へ遷移
- 人気記事TOP5: **現在のページのブログデータ**からいいね数順で上位5件を算出（全ブログの全体TOP5ではない）
- タグ一覧: GlobalContextから取得、クリックで `/tag/:tag` へ遷移

### Footer

- サイト概要
- リンク（お問い合わせ）
- SNSリンク（Twitter, GitHub）
- コピーライト

## 10. 通知（Toast）

react-hot-toastを使用。全操作の成功/失敗をトーストで通知。

| カテゴリ | 成功メッセージ | エラーメッセージ |
|---------|--------------|----------------|
| ログイン | ログインしました | メールアドレスまたはパスワードが正しくありません |
| ログアウト | ログアウトしました | ログアウトに失敗しました |
| 記事作成 | ブログを作成しました | ブログの作成に失敗しました |
| 記事更新 | ブログを更新しました | ブログの更新に失敗しました |
| 記事削除 | ブログを削除しました | ブログの削除に失敗しました |
| いいね | ブログをいいねしました | ブログのいいねに失敗しました |
| いいね取消 | ブログのいいねを取り消しました | ブログのいいねの取り消しに失敗しました |
| コメント | コメントを作成しました | コメントの作成に失敗しました |
