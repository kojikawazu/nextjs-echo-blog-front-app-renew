# 要件定義書（Requirements Specification）

## 1. 機能要件一覧

### FR-01: ブログ記事一覧表示

| 項目 | 内容 |
|------|------|
| 優先度 | 高 |
| 説明 | ブログ記事をカード形式で一覧表示する |
| 受入条件 | 記事のタイトル、カテゴリ、タグ、作成日、いいね数が表示される |

- ページネーション対応（1ページあたり10件）
- 新着順/人気順のソート切り替え
- カテゴリ・タグによるフィルタリング
- キーワード検索（タイトル、説明文、タグを対象）
- 検索にデバウンス（300ms）を適用

### FR-02: ブログ記事詳細表示

| 項目 | 内容 |
|------|------|
| 優先度 | 高 |
| 説明 | 個別のブログ記事を詳細表示する |
| 受入条件 | タイトル、タグ、作成日、本文（Markdown）、コメント欄が表示される |

- GitHub URLが設定されている場合、GitHub API経由でMarkdownを取得・レンダリング
- Markdownはシンタックスハイライト（highlight.js）、GFM、改行対応
- 記事オーナーの場合、編集リンクを表示

### FR-03: ユーザー認証

| 項目 | 内容 |
|------|------|
| 優先度 | 高 |
| 説明 | メールアドレスとパスワードによるログイン/ログアウト |
| 受入条件 | Cookie認証でセッションが維持される |

- ログインフォーム（メール + パスワード）
- 新規登録フォーム（名前 + メール + パスワード）**※UIのみ実装、バックエンドAPI未連携（スタブ）**
- 認証状態チェック（ページ読み込み時に自動実行）
- ログイン成功後、ホーム画面へリダイレクト
- ログアウト後、ログイン画面へリダイレクト

### FR-04: 記事作成

| 項目 | 内容 |
|------|------|
| 優先度 | 高 |
| 説明 | 認証ユーザーが新しいブログ記事を作成する |
| 受入条件 | フォーム入力後、確認モーダルを経て記事が作成される |

- 入力項目: タイトル、GitHub URL、カテゴリ、タグ、内容
- Zodバリデーション + React Hook Formによるフォーム管理
- 確認モーダルで作成確認
- 未認証ユーザーはホームへリダイレクト

### FR-05: 記事編集

| 項目 | 内容 |
|------|------|
| 優先度 | 高 |
| 説明 | 認証ユーザーが自身のブログ記事を編集する |
| 受入条件 | 既存のデータがフォームに初期値として表示される |

- 記事オーナーのみ編集可能（非オーナーはログイン画面へリダイレクト）
- 更新確認モーダル
- 更新成功後、記事詳細ページへリダイレクト

### FR-06: 記事削除

| 項目 | 内容 |
|------|------|
| 優先度 | 高 |
| 説明 | 認証ユーザーが自身のブログ記事を削除する |
| 受入条件 | 削除確認モーダルを経て記事が削除される |

- 編集画面内の削除ボタンから実行
- 削除確認モーダル
- 削除成功後、ホームへリダイレクト

### FR-07: いいね機能

| 項目 | 内容 |
|------|------|
| 優先度 | 中 |
| 説明 | ゲストユーザーを含む全ユーザーが記事にいいねできる |
| 受入条件 | 訪問者IDで同一ユーザーのいいね重複を防止する |

- 訪問者IDの自動生成（バックエンドAPI経由）
- いいね登録/取り消しのトグル操作
- いいね済み記事の判定
- TanStack Queryによるキャッシュ無効化で即時反映

### FR-08: コメント機能

| 項目 | 内容 |
|------|------|
| 優先度 | 中 |
| 説明 | ゲストユーザーが記事にコメントを投稿できる |
| 受入条件 | ゲスト名とコメント内容を入力して投稿可能 |

- ゲストユーザー名 + コメント本文
- リプライ機能は型定義（`parent_id`）のみ存在し、UIおよびAPI送信は未実装
- TanStack Queryによるキャッシュ無効化で即時反映

### FR-09: サイドバー

| 項目 | 内容 |
|------|------|
| 優先度 | 中 |
| 説明 | カテゴリ、人気記事、タグの一覧をサイドバーに表示 |
| 受入条件 | GlobalContextからデータを取得して表示する |

- カテゴリ一覧（クリックでカテゴリフィルタページへ遷移）
- 人気記事TOP5（いいね数順）
- タグ一覧（クリックでタグフィルタページへ遷移）

### FR-10: GitHub Markdownプロキシ

| 項目 | 内容 |
|------|------|
| 優先度 | 高 |
| 説明 | GitHub上のMarkdownファイルをサーバーサイドで取得しクライアントに返す |
| 受入条件 | 許可されたオーナーのリポジトリのみアクセス可能 |

- gray-matterでフロントマターを除去し、本文のみ返却
- URL形式バリデーション（`https://github.com/{owner}/{repo}/blob/main/{path}.md`）
- ALLOWED_REPO_OWNERによるアクセス制御

## 2. 非機能要件概要

| カテゴリ | 要件 |
|---------|------|
| パフォーマンス | 検索にデバウンス（300ms）を適用し不要なAPIコールを抑制 |
| セキュリティ | BFFプロキシでバックエンドURLを秘匿、Secret Managerで機密情報管理 |
| 可用性 | Cloud Run + Cloudflareによる高可用性構成 |
| 保守性 | TypeScript、Zod、ESLint、Prettierによるコード品質担保 |
| テスト | Playwright E2Eテストによるリグレッション防止 |

## 3. 画面一覧

| パス | 画面名 | 認証要否 | 説明 |
|------|--------|---------|------|
| `/` | ホーム | 不要 | ブログ一覧（検索・フィルタ・ソート・ページネーション） |
| `/blog/[id]` | ブログ詳細 | 不要 | 記事詳細表示 + コメント |
| `/category/[category]` | カテゴリ別一覧 | 不要 | 特定カテゴリの記事一覧 |
| `/tag/[tag]` | タグ別一覧 | 不要 | 特定タグの記事一覧 |
| `/new` | 記事作成 | 必要 | 新規記事作成フォーム |
| `/edit/[id]` | 記事編集 | 必要（オーナーのみ） | 記事編集・削除フォーム |
| `/login` | ログイン | 不要 | ログインフォーム |
| `/register` | 新規登録 | 不要 | ユーザー登録フォーム（UIのみ、API未連携） |
